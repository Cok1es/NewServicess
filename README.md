Отчет по реализации микросервисной архитектуры
Выполненные задачи:
1. Миграция системы сборки:
  1.1 Осуществил полный переход проекта с Maven на Gradle.
  1.2 Настроил многомодульную сборку, оптимизировал управление зависимостями и сократил время сборки артефактов.
2. Стабилизация и разрешение конфликтов (Dependency Management):
  2.1 Успешно устранил критические конфликты библиотек (Jar Hell) при переходе на Spring Boot 3.2.4 и Java 21.
  2.2 Разрешил специфические ошибки автоконфигурации (RestClient, LoadBalancer) и обеспечил совместимость компонентов Spring Cloud.
3. Развертывание инфраструктурных паттернов:
  3.1 Внедрил и настроил Service Discovery (Netflix Eureka) для динамической регистрации сервисов.
  3.2 Реализовал Externalized Configuration (Spring Cloud Config) для централизованного управления настройками всех микросервисов.
  3.3 Настроил API Gateway как единую точку входа с маршрутизацией запросов.
4. Контейнеризация и оркестрация (Docker):
  4.1 Разработал оптимальные Docker-файлы для каждого микросервиса на базе легковесных образов (JRE Alpine).
  4.2 Создал комплексный docker-compose.yml, объединяющий 5 микросервисов и внешнюю инфраструктуру (PostgreSQL, Kafka, Zookeeper).
  4.3 Обеспечил корректное взаимодействие сервисов внутри изолированной Docker-сети через имена контейнеров.
Результат:
Полностью работоспособная, масштабируемая микросервисная система, которая разворачивается одной командой docker-compose up --build.
